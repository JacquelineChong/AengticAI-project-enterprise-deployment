7. Scaling Governance to Different Project Types

After establishing our data lake foundation, we need a clear framework for classifying and managing different types of data projects. Our organization will undertake initiatives ranging from strategic, enterprise-wide transformations to routine operational reporting tasks. Each requires different levels of governance, stakeholder involvement, and DCoE support to succeed.

Three Types of Data Projects

Type 1: Transformative Projects

12-18 months

Description: Strategic, enterprise-wide initiatives that fundamentally change how the organization operates with data.

Examples:

  • Implementation of enterprise-wide data lake
  • Integrated inventory management platform
  • Customer 360Â° view across all touchpoints

End Users: Multiple internal departments, potentially external customers

Governance: Formal with steering committee and stage-gate approach

Type 2: Cross-Departmental Projects

3-4 months

Description: Tactical initiatives spanning multiple business units with focused scope.

Examples:

  • Seasonal menu optimization (Operations + Marketing)
  • Supplier performance analytics (Procurement + Finance)
  • Store traffic analysis (Operations + Sales)

End Users: 2-3 internal departments

Governance: Semi-formal with lightweight approvals and weekly reviews

Type 3: Operational Data Tasks

Ongoing/Recurring

Description: Regular, recurring activities supporting day-to-day business operations.

Examples:

  • Daily/weekly inventory dashboards
  • Monthly financial reports
  • Regular customer behavior insights

End Users: Specific department or functional team

Governance: Minimal with standard templates and automated quality checks

Data Product/ Project Development Approach

We recommend adapting a "6S" data product development framework for all project types. This approach provides a structured methodology that scales from complex transformative initiatives to routine operational tasks, ensuring consistency while allowing for appropriate flexibility.

Source: Rewired: The McKinsey Guide to Outcompeting in the Age of Digital and AI

6S Approach for Transformative Projects (Type 1)

1
Scoping

Focus: Comprehensive enterprise scoping with executive alignment

Activities:

  • Executive workshops to identify strategic objectives
  • Cross-functional value assessment
  • Current state analysis across all affected domains
  • Vision development with 2-3 year horizon
  • Resource planning for extended implementation

DCoE Involvement: Very High

Deliverables: Strategic roadmap, Value case, Resource plan, Timeline

2
Selecting

Focus: Enterprise-wide data selection with future-proofing

Activities:

  • Enterprise data mapping across all systems
  • Data prioritization based on strategic value
  • Technical debt assessment
  • Data lineage documentation
  • Long-term data retention planning

DCoE Involvement: Very High

Deliverables: Enterprise data catalog, Data dictionary, Metadata strategy

3
Sourcing

Focus: Comprehensive data acquisition from diverse sources

Activities:

  • Data profiling across all source systems
  • Quality assessment with detailed metrics
  • ETL/ELT pipeline development for multiple sources
  • Master data management implementation
  • Data reconciliation procedures

DCoE Involvement: High

Deliverables: Data quality framework, ETL/ELT architecture, Integration patterns

4
Structuring

Focus: Enterprise data architecture with flexibility for future needs

Activities:

  • Enterprise data modeling (conceptual, logical, physical)
  • Multi-layer data architecture (raw, standardized, consumption)
  • Data governance framework implementation
  • Scalable storage solutions
  • Performance optimization for diverse use cases

DCoE Involvement: Very High

Deliverables: Enterprise data model, Technical architecture, Governance framework

5
Sharing

Focus: Multi-channel delivery platform for enterprise insights

Activities:

  • Self-service analytics platform development
  • Enterprise dashboard framework
  • API strategy for internal/external consumption
  • Data visualization standards
  • Training program for business users

DCoE Involvement: High

Deliverables: Analytics platform, Visualization library, Data API catalog

6
Steering

Focus: Enterprise governance structure and continuous enhancement

Activities:

  • Data governance council establishment
  • KPI framework for ongoing evaluation
  • Operating model for data management
  • Long-term staffing and capability building
  • Business adoption program

DCoE Involvement: Very High

Deliverables: Governance procedures, RACI matrix, Enhancement roadmap

6S Approach for Cross-Departmental Projects (Type 2)

1
Scoping

Focus: Department-level scoping with key stakeholders

Activities:

  • Department leader workshops (2-3 departments)
  • Business case development for specific pain points
  • Prioritization of business requirements
  • 3-4 month implementation planning
  • Success metrics definition

DCoE Involvement: Medium

Deliverables: Project charter, Requirements document, Success criteria

2
Selecting

Focus: Targeted data selection from relevant systems

Activities:

  • Data mapping focused on 2-3 key systems
  • Identification of critical data elements
  • Relevant historical data assessment
  • Gap analysis for required data elements
  • Reuse of existing enterprise data assets

DCoE Involvement: Medium

Deliverables: Data inventory, Gap analysis, Source mapping

3
Sourcing

Focus: Efficient data acquisition from departmental systems

Activities:

  • Targeted data profiling
  • Department-specific quality rules
  • Leveraging existing data pipelines
  • Department data steward engagement
  • Data reconciliation between departments

DCoE Involvement: Medium

Deliverables: Data quality assessment, ETL workflow, Quality rules

4
Structuring

Focus: Solution-specific data model aligned with enterprise standards

Activities:

  • Solution-specific data modeling
  • Alignment with enterprise data models
  • Performance optimization for specific use cases
  • Implementation of departmental business rules
  • Data quality controls for critical metrics

DCoE Involvement: Medium

Deliverables: Solution data model, Business rules documentation

5
Sharing

Focus: Department-specific insights with cross-functional relevance

Activities:

  • Department dashboard development
  • Cross-functional reporting views
  • Scheduled data delivery setup
  • Role-based access implementation
  • Training for department users

DCoE Involvement: Low

Deliverables: Department dashboards, Cross-functional reports, Training guide

6
Steering

Focus: Project-specific governance with departmental ownership

Activities:

  • Department data owners assignment
  • Cross-functional coordination process
  • Weekly project status reviews
  • Value tracking against success metrics
  • Knowledge transfer to operational teams

DCoE Involvement: Low

Deliverables: Project RACI, Operational handover plan, Value realization report

6S Approach for Operational Data Tasks (Type 3)

1
Scoping

Focus: Template-based scope definition for recurring needs

Activities:

  • Standard requirements template completion
  • Business purpose documentation
  • Identification of data consumers
  • Frequency and timing requirements
  • Categorization of information needs

DCoE Involvement: Very Low

Deliverables: Requirements form, User list, Delivery schedule

2
Selecting

Focus: Standard data sets with predefined elements

Activities:

  • Selection from standard data catalog
  • Confirmation of data availability
  • Identification of relevant time periods
  • Filter and parameter specification
  • Data element inclusion/exclusion

DCoE Involvement: None

Deliverables: Data specification, Parameter configuration

3
Sourcing

Focus: Automated data collection from established sources

Activities:

  • Use of predefined data connectors
  • Scheduled data refresh setup
  • Standard data quality checks
  • Exception handling configuration
  • Alert thresholds for data issues

DCoE Involvement: None

Deliverables: Refresh schedule, Quality check configuration, Alert setup

4
Structuring

Focus: Use of predefined data structures and patterns

Activities:

  • Selection of standard data model template
  • Application of department-specific business rules
  • Configuration of calculation logic
  • Utilization of standard dimensions/hierarchies
  • Implementation of common aggregations

DCoE Involvement: Very Low

Deliverables: Data structure configuration, Calculation definitions

5
Sharing

Focus: Regular production of standard outputs

Activities:

  • Use of standard dashboard templates
  • Scheduled report distribution
  • User access management
  • Export format configuration
  • Notification setup for data consumers

DCoE Involvement: Very Low

Deliverables: Operational dashboard, Scheduled reports, Distribution list

6
Steering

Focus: Standard operating procedures for ongoing maintenance

Activities:

  • Department data owner assignment
  • Data quality monitoring
  • Standard review cycles
  • Issue resolution process
  • User feedback collection

DCoE Involvement: Very Low

Deliverables: Operational procedures, Support process, Feedback mechanism
